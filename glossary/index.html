§ 00 — Overview

The ML pipeline, mapped.

This glossary covers the four stages of the ML lifecycle — from raw data handling through model training, evaluation, and production deployment. Each section maps to a stage in the diagram below.

§ 01 — Data Handling

Data handling.

Techniques for preprocessing, transforming, and engineering features from raw data before feeding it into a model.

13 terms

0-1 Scaling
A preprocessing technique that transforms feature values to all fall in the range of [0, 1]. Similar to standardization, this also helps machine learning algorithms learn by keeping features on comparable scales.
Akaike Information Criterion AIC
A way of evaluating how well a model fits a set of data while also balancing model complexity. Lower AIC values indicate a "better" model — the score is lower if the model has fewer parameters and a higher likelihood of having produced the data. Used for model selection when there are multiple candidates.
Bayes Information Criterion BIC
A way of evaluating and selecting models, similar to the AIC. The main practical difference is that BIC applies a stronger penalty to more complex models, especially as sample size increases.
Bucketing pd.cut
A type of feature engineering that bins continuous values into discrete buckets. For example, a feature representing someone's age could be bucketed into age ranges. These buckets can capture important information about patterns in the data that a raw continuous value may obscure.
Count Vectorizer
A way of feature-engineering unstructured text data. It iterates through a corpus of text to construct a vocabulary of all possible tokens. Every token is assigned an index. A piece of text is then converted to a vector where each index represents the number of occurrences of the corresponding token in that text.
Missing Data
Values not present for certain features for some examples in your dataset. Many ML algorithms require all features to have defined values. Standard handling techniques:
  • Remove: Remove examples that have missing feature values. Can also remove features entirely if too few examples have values for them.
  • Fill in — Means: Fill in missing values with the average of all non-missing values for that feature.
  • Fill in — Linear interpolation: For ordered data (e.g. time series), create a line between surrounding data points and use it to fill in missing values.
  • Fill in — Clustering & similarities: Assign examples with missing data to clusters based on similarity, then fill with statistics from that cluster (e.g. the cluster mean).
One-Hot Encoding
A way of handling categorical data for models that require numerical features. It creates a separate binary column for each possible category value, assigning 1 to the column matching the example's category and 0 to all others.
Percentile Interpolation
Inferring missing values using values drawn from the percentiles of the feature's distribution (e.g. the 25th, 50th, or 75th percentile).
Remove Outliers
In ML, outliers are examples that are not representative of general patterns in the data. Removing them can help models learn patterns aligned with the majority of training data. However, this should be done with care — outliers may contain important information depending on why they occur.
Standardization
A preprocessing technique that transforms feature values such that the distribution has a mean of 0 and a standard deviation of 1. Keeping features on the same scale helps many ML algorithms learn patterns more effectively.
TF-IDF Vectorizers Term Frequency–Inverse Document Frequency
A modification of count vectorization that weights token frequencies by how often they appear across documents, to gauge the relative importance of each term. TF-IDF multiplies the term frequency (relative frequency within a document) by the inverse document frequency (log inverse of how many documents contain the term). Terms with high frequency within a document but low frequency across the corpus receive a higher score.
Transformations
Operations applied to input data to correctly prepare it for a machine learning model. Transformations can be applied to numerical, categorical, and text data. Examples include scaling, encoding, and tokenization.
Word Embeddings
A way of feature-engineering unstructured text to incorporate semantic meaning. Each word is assigned a learned vector (an embedding), trained to represent information about the word in context. Embeddings are typically a component of a model trained on a meaningful task, such as predicting the next word in a sequence.
§ 02 — Model & Optimization

Model selection & optimization.

Algorithms, architectures, and training techniques for building models that generalize well to unseen data.

52 terms

Activation Functions
Functions applied within neural networks after a layer's linear transformation. Used to introduce nonlinearity into the function the network computes, enabling it to learn complex patterns beyond linear relationships.
Area Under the ROC Curve AUC
A classification metric measuring the total area under the ROC curve. A larger area indicates that changes in threshold lead to a larger change in true positive rate relative to false positive rate — meaning better overall performance across thresholds.
Autoregressive Integrated Moving Average ARIMA
A common statistical model used for forecasting time series data. Combines autoregression (using past values to predict future ones), differencing (to achieve stationarity), and a moving average component.
Bias-Variance Tradeoff
The expected generalization error of an ML model can be decomposed into bias (the difference between the expected prediction and the true value) and variance (the spread or instability of predictions across different training sets). Methods that reduce bias often increase variance, and vice versa.
Classification Method
A way of assigning examples to discrete categories given input data. Classification algorithms learn a decision boundary from labeled training examples and use it to predict the class of new inputs.
Coefficient Interpretation
One of the main benefits of linear regression. Learned coefficients indicate the expected change in the output for a one-unit increase in the corresponding input feature, holding all other features constant, which allows gauging the relative importance of each feature.
Cosine Similarity
A similarity measure for two vectors computed as their dot product divided by the product of their magnitudes — mathematically equivalent to the cosine of the angle between them. Results range from -1 (opposite directions) to 1 (same direction); 0 means the vectors are orthogonal.
Crop
An image preprocessing technique that takes a sub-region of the image as a training data point. Commonly used in computer vision because it can help a model focus on specific parts of an image and implicitly augments the training dataset.
Deep Learning
A subfield of machine learning focused on neural networks with many layers. Deep networks can learn hierarchical representations of data, enabling state-of-the-art performance on tasks such as image recognition, language modeling, and speech recognition.
Dropout
A regularization technique for training neural networks where nodes are randomly dropped (set to zero) on each forward pass. This reduces the capacity of the network during training and helps prevent overfitting by preventing co-adaptation of features.
Ent (Error-based splitting)
A method of determining the best split point for a node in a decision tree. It calculates the probability of incorrectly classifying an example based on all examples in a node, guiding the tree to split in ways that minimize classification error.
Epochs
The number of complete passes through the entire training dataset during model training. One epoch means the model has seen every training example once. Multiple epochs allow the model to refine its parameters iteratively.
Entropy
A measure of impurity or disorder in a set of data. In decision trees, entropy is calculated based on the proportions of elements in each class. A node with equal proportions of all classes has maximum entropy; a pure node (all one class) has zero entropy.
Euclidean Distance
The straight-line distance between two points — the same as the distance formula from 2D algebra, generalized to vectors of any dimension. Commonly used as a similarity or proximity measure in clustering, KNN, and other algorithms.
Exp() — Exponential Function
Used in many ML activation functions, particularly sigmoid and softmax. Properties that make it useful include producing a smooth, always-positive curve and being convenient for gradient-based optimization because its derivative equals itself.
Feature Importance
With tree-based methods, the relative importance of each feature can be computed by pooling the split criteria values (e.g. decrease in Gini impurity) for that feature across all trees in an ensemble. Higher aggregate values indicate features that contribute more to predictive performance.
Gini Index
A method of determining the best split point for a node in a decision tree. It calculates the probability of incorrectly classifying a randomly chosen element if it were randomly labeled according to the distribution of labels in the node. A Gini index of 0 indicates a pure node.
Gradient Boosting
An ensemble method that combines predictions from multiple weak learners (typically decision stumps — trees of height 1), trained in a stagewise fashion. Each subsequent tree learns to predict the residual errors of the previous ensemble, using the gradient of a chosen loss function to move predictions closer to true values.
Gradient Method
An optimization technique that uses the gradient of a function with respect to its parameters to take iterative steps toward a local or global minimum. Gradient descent is the most common example: parameters are updated in the direction opposite to the gradient.
Hierarchical Clustering
A type of clustering that organizes clusters into a hierarchy (a dendrogram). Can be constructed bottom-up (agglomerative: merging smaller clusters into larger ones) or top-down (divisive: splitting large clusters). Does not require specifying the number of clusters in advance.
Information Gain
A method of determining the best split point for a decision tree node. Information gain measures the reduction in entropy achieved by splitting the data on a given feature. It is calculated as the entropy of the parent node minus the weighted entropy of the resulting child nodes.
Inverse Optimization
An advanced concept that, unlike standard optimization which minimizes a known objective, works backward from observed decisions or outcomes to infer the objective function that would make those decisions optimal. Used in settings where the underlying goals are unknown but their outputs are observable.
K-Means Clustering
An unsupervised algorithm that assigns data points to k centroids. Centroids are initially set to random values, then iteratively updated: each point is assigned to its nearest centroid, and each centroid is recomputed as the mean of all points assigned to it. Repeats until centroids stop moving (convergence).
Lasso Regression L1 Regularization
A modification of linear regression that adds the L1 norm of the weights to the loss function as a penalty. This drives some coefficients exactly to zero, effectively performing variable selection. Used for regularization and feature selection simultaneously.
Layers
Subunits in a neural network composed of interconnected nodes. Implemented as matrix multiplications. Different layer types (dense, convolutional, recurrent, attention) perform these multiplications in different ways, each suited to different data structures and tasks.
Linear Regression
An ML algorithm that models the relationship between inputs and output as the line (or hyperplane) of best fit — the linear equation that minimizes mean squared error. Produces interpretable coefficients and is a foundation for many more complex methods.
Log Odds
Defined as the natural logarithm of p / (1 − p). Most relevant to logistic regression: rearranging the model equation shows that a linear combination of input features equals the log odds of the positive class. Increasing a feature by 1 increases the log odds by the corresponding coefficient.
Logistic Function
The generalized version of the sigmoid function. It squashes real-valued inputs to the range (0, 1), making it suitable for outputting probabilities. Used as the final activation in binary classification models and as a component of the softmax for multi-class problems.
Logistic Regression
A parametric model for classification. Takes a linear combination of input features and passes it through an activation function (sigmoid for binary, softmax for multi-class) to output predicted probabilities. Despite the name, it is a classification algorithm, not a regression one.
Loss Functions
Functions that compute the error between predicted values and actual values. During training, the optimizer adjusts model weights to minimize the loss on the training data. Common examples: mean squared error (regression), cross-entropy (classification).
Manhattan Distance
Measures the distance between two points as the sum of the absolute differences between their respective coordinate values — analogous to navigating a city grid. Can be calculated for vectors of any dimension. Sometimes preferred over Euclidean distance in high-dimensional spaces.
Nodes
Single units in a neural network. Each node receives a weighted sum of inputs from connected incoming nodes, applies an activation function to that sum, and passes the result to connected outgoing nodes. The collection of weighted connections defines the network's learned function.
Optimizers
Methods that speed up or improve the training process in neural networks. One common example is momentum, which computes the current gradient update as a rolling average of past and present gradients, helping to dampen oscillations and accelerate convergence. Other examples include Adam and RMSProp.
P-Value Interpretation
The probability of observing a test statistic as extreme as (or more extreme than) the one calculated, under the assumption that the null hypothesis is true. A small p-value (typically < 0.05) suggests the observed result is unlikely under the null hypothesis.
Principal Component Analysis PCA
An unsupervised dimensionality reduction technique that transforms original features into a set of orthogonal components that capture the maximum variance in the data. The components are ordered so the first retains the most variance, allowing lower-dimensional representations with minimal information loss.
Prophet
A forecasting framework written by Meta that simplifies modeling time series data. Provides built-in functionality for trend forecasting, seasonality modeling (yearly, weekly, daily), and evaluation. Designed to handle common time series quirks with minimal configuration.
PyTorch
A Python framework for designing and training deep neural networks. Implements a tensor library with GPU acceleration, automatic gradient computation, backpropagation, and high-level abstractions for network layers, optimizers, and data loading. Increasingly supports distributed training.
Random Forests
An ensemble algorithm that combines predictions from multiple decision trees to improve generalization. Each tree is trained on a random subset of features and examples, reducing the tendency to overfit. The ensemble averages predictions (regression) or takes a majority vote (classification).
Receiver Operating Characteristic Curve ROC
A curve that plots the true positive rate (TPR / recall) on the y-axis against the false positive rate (FPR = FP / total negatives) on the x-axis at various classification thresholds. Used to gauge overall classifier performance independent of any single threshold choice.
Regularization
Techniques used to reduce overfitting on training data and improve performance on unseen data. Applied when a model fits training data too closely and generalizes poorly. Common forms include L1 (Lasso), L2 (Ridge), and dropout for neural networks.
Resize
An image preprocessing step that changes the spatial dimensions of images in a dataset. Necessary because trained neural network models have a fixed input size — the number of input pixels must match the number of parameters in the input layer.
Ridge Regression L2 Regularization
A variant of linear regression that adds the L2 norm of the weights to the loss function as a penalty. Unlike Lasso, this shrinks coefficients toward zero but rarely to exactly zero. Used for regularization and can also be used for variable selection.
Seasonality
In time series data, predictable periodic changes expected to repeat at a fixed frequency — most commonly yearly. For example, retail sales spiking during the holidays or energy usage peaking in summer. Seasonality must be modeled or removed for accurate forecasting.
SHAP Values SHapley Additive exPlanations
A concept based in cooperative game theory used to compute feature importance. Models features as "players" and predictions as "payout." SHAP provides instance-level explanations, quantifying each feature's contribution to a specific prediction rather than just global importance.
Single Decision Tree
A model that uses a series of hierarchical if-then decisions to iteratively partition the training data and predict outcomes. Each internal node represents a feature split, each branch a decision, and each leaf a predicted value or class. Interpretable but prone to overfitting.
Stepwise Regression
A method of selecting variables for a regression model in a series of steps. On each iteration, variables are added or removed based on statistical significance (p-values or information criteria), then a new model is fit. Repeats until a stopping criterion is met.
Supervised Learning
A type of machine learning that predicts an outcome based on labeled input-output pairs. The algorithm learns to detect relationships between inputs and known outcomes during training, which it then uses to make predictions on new, unseen inputs.
The Five Assumptions of Linear Regression
Linear regression is valid when the following five conditions hold:
  1. Linearity: A linear relationship exists between input variables and the outcome.
  2. Independence: Observations are independent of each other.
  3. Normality: Residuals are normally distributed.
  4. Homoscedasticity: Residuals have constant variance across all input values.
  5. No autocorrelation: Residuals are independent of each other (no temporal or spatial patterns).
When these assumptions are violated, predicted outcomes may be incorrect, coefficient interpretations may be misleading, and confidence intervals may be inaccurate.
Tree-Based Models
Machine learning models based on decision trees, which use hierarchical if-then splits to partition training data and make predictions. Common splitting criteria include the Gini index and entropy (from information theory). Examples include single decision trees, random forests, and gradient boosting.
Unsupervised Learning
A type of machine learning that finds patterns based only on the features of the input, without using associated outcome labels. Common examples include clustering (grouping similar examples) and dimensionality reduction (finding compact representations).
Variable Selection
Choosing which input variables to include in a model based on specific criteria — typically whether those variables are predictive of the outcome. Methods include statistical tests, regularization-induced sparsity (Lasso), stepwise regression, and tree-based feature importance.
Variable Usefulness
Supervised ML algorithms often provide ways to measure which input variables are most important for predicting outcomes — for example, coefficient magnitude in linear models or feature importance in tree-based models. Assessing usefulness guides variable selection and statistical analysis.
§ 03 — Evaluation

Evaluation methods & metrics.

Metrics and methods for measuring how well a trained model performs, detecting failure modes, and comparing alternative models.

9 terms

Accuracy
A classification metric: the total number of correctly predicted examples divided by the total number of examples. Not suitable for class-imbalanced datasets — a model predicting all examples as the majority class can achieve high accuracy while being useless for detecting the minority class.
Area Under the ROC Curve AUC
A classification metric measuring the total area under the ROC curve. A larger area indicates that changes in threshold lead to a larger change in true positive rate compared to false positive rate — meaning better overall discriminative performance across all thresholds.
Class Imbalance
A difference in the proportions of each class in training data. Training on imbalanced data can produce models that only perform well on frequent classes. Handling strategies include oversampling (e.g. SMOTE), undersampling, and cost-sensitive learning, though results are empirically mixed and dataset-dependent.
F1 Score
A classification metric that computes the harmonic mean of precision and recall: 2 × (precision × recall) / (precision + recall). Useful when both false positives and false negatives are costly, or when class imbalance makes accuracy misleading.
Mean Absolute Error MAE
A regression metric that measures the mean of the absolute differences between predicted and actual values. More appropriate than MSE when data contains outliers, since it does not square the errors — extreme errors are not disproportionately amplified.
Mean Squared Error MSE
A regression metric measuring the average of the squared differences between predicted and actual values. Squaring errors penalizes large deviations more heavily than small ones, making MSE sensitive to outliers. Its gradient is smooth, which makes it convenient for gradient-based optimization.
Precision
A classification metric: how many of the examples predicted as positive were actually positive. Calculated as TP / (TP + FP). High precision means the model rarely calls something positive when it is not. Critical when false positives are costly (e.g. spam filtering, medical diagnosis).
Receiver Operating Characteristic Curve ROC
A curve plotting true positive rate (TPR = recall) on the y-axis against false positive rate (FPR = FP / total negatives) on the x-axis at various classification thresholds. Sweeping from threshold 1.0 to 0.0 traces the curve. An ideal classifier hugs the top-left corner.
Recall Sensitivity · True Positive Rate
A classification metric: how many of the actual positive examples the model correctly identified. Calculated as TP / (TP + FN). High recall means the model misses few positives. Critical when false negatives are costly (e.g. disease screening, fraud detection).
§ 04 — Production

ML in production.

Strategies and concepts for deploying, monitoring, and maintaining machine learning models in live systems at scale.

19 terms

A/B Testing
An experimentation process used to compare two model versions. A control group receives the existing model and a test group receives the new model; a metric of interest is measured for each. Hypothesis testing determines whether the difference is statistically significant. The most common method for evaluating models before full deployment.
Bandits
Algorithms that balance exploration (trying different options with unknown potential rewards) and exploitation (continuing with options known to perform well). In ML, bandit algorithms can be used to evaluate and select between models in production, adapting traffic allocation based on observed performance.
Batch Prediction
A type of model inference that runs asynchronously — not in direct response to a user request. Triggered as a scheduled job or pipeline step. Useful for precomputing results (e.g. user embeddings, recommendations) where features are known in advance and real-time latency is not required.
Canary Release
A deployment technique for evaluating a new model without fully impacting users. The new model runs in parallel with the existing one; a small portion of traffic is routed to it. Traffic is increased gradually depending on performance. Allows early detection of issues before full rollout.
Concept Drift
A type of distribution shift in which the relationship between input features and the target variable changes over time — P(Y | X) shifts. The model's learned decision boundary no longer reflects the true underlying process, leading to degraded performance even if input distributions are stable.
Containers
A unit of software that encapsulates source code, dependencies, and configuration for a service into a portable, isolated environment. Containers (e.g. Docker) make deployments reproducible and isolate ML services from other parts of a system.
Covariate Shift
A type of distribution shift in which the distribution of the model's input features changes between training and deployment — P(X) shifts — while the conditional relationship P(Y | X) remains the same. Can cause model predictions to degrade even if the model's learned function is still correct.
Edge Computing
Moving computations to the "edge" — running inference on the consumer device (mobile phone, laptop, embedded device) rather than on remote servers. Reduces latency, network usage, and privacy concerns, but requires model compression to fit hardware constraints.
Feature Flags
Configuration switches used to enable or disable features or models in production without deploying new code. Useful for A/B testing (directing traffic to different model versions) and canary releases (gradually increasing exposure to a new model).
Kubernetes
An open-source container orchestration platform for deployment and scaling. Can efficiently manage compute resources, auto-scale services based on load, create complex multi-container applications, and monitor running services. Widely used for deploying ML inference services.
Label Shift
A type of distribution shift in which the distribution of the model's target variable (labels) changes between training and deployment — P(Y) shifts — while P(X | Y) remains stable. Common in domains where the prevalence of a class changes over time (e.g. a disease becoming more or less common).
Model Compression
Techniques for making a model smaller to reduce inference latency and memory footprint. Common methods include:
  • Low-rank factorization: Decomposing large weight matrices into smaller ones.
  • Quantization: Reducing the numerical precision of weights (e.g. float32 → int8).
  • Knowledge distillation: Training a smaller "student" model to mimic a larger "teacher" model.
Model Drift
Degradation of a model's production performance over time, caused by changes in the relationship between input features and the target variable. A broader term encompassing concept drift, covariate shift, and label shift. Requires regular monitoring and retraining to address.
Model Retraining
Training a model further to restore or improve performance that has degraded in production. Can be done offline on a fixed dataset or in an automated pipeline that continuously ingests new labeled data. The trigger is typically a detected drop in a monitored production metric.
Monitoring
Continuously watching metrics and behavior of a deployed ML model. Includes tracking distributions of input features and predictions over time, measuring inference latency, and flagging anomalies that might indicate drift or bugs. Essential for maintaining model quality after deployment.
Prediction Service
A component of a system or pipeline that serves predictions from a trained ML model. Receives feature inputs, runs inference through the model, and returns predictions. Can serve real-time requests (low latency) or batch jobs (high throughput).
Real-Time Prediction
A type of model inference served quickly in direct response to a request — typically with latency requirements in the milliseconds-to-seconds range. Examples include search ranking, recommendation retrieval, and fraud detection at transaction time.
Shadow Deployment
A deployment technique for evaluating a new model without impacting users. The new model runs in parallel with the production model; predictions are logged but not served to users. Allows safe comparison of model outputs without any risk to the user experience.
Telemetry Data
The collection of data, logs, and metrics from a running system, used for monitoring and debugging. In ML systems, telemetry includes prediction logs, input feature distributions, latency measurements, and error rates. Essential for detecting drift and diagnosing production issues.
