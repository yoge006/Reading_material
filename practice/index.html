12 Questions
NLP
01
Explain the bottleneck problem in seq2seq models and how attention solves it.
NLP Medium
›

In a vanilla seq2seq model, the encoder processes the entire input sequence and compresses it into a single fixed-length vector — the final hidden state. The decoder then generates the output from this one vector. For short inputs this works fine, but for long sequences the single vector can't hold all the relevant information; early tokens get diluted or forgotten as the encoder processes more of the sequence.

Bahdanau attention solves this by letting the decoder dynamically attend to different encoder hidden states at each step. Instead of one fixed context vector, the decoder computes a fresh context vector cᵢ for each output position:

  • Score each encoder state hⱼ against the previous decoder state sᵢ₋₁ using a learned score function
  • Normalize the scores via softmax to get attention weights αᵢⱼ
  • Compute cᵢ = Σⱼ αᵢⱼ · hⱼ — a weighted sum that focuses on the relevant source positions

The decoder no longer has to read everything from a single vector. It has direct, differentiable access to all encoder states, weighted by relevance.

02
What is the difference between Bahdanau (additive) and Luong (multiplicative) attention?
NLP Architecture Hard
›

Bahdanau (additive) attention uses a learned feed-forward network to compute alignment scores:
e_ij = vᵀ tanh(W₁s_{i-1} + W₂h_j). It introduces parameters W₁, W₂, and v. The score is computed before the decoder generates the current output token (it uses the previous decoder state s_{i-1}).

Luong (multiplicative) attention computes scores after the decoder generates a hidden state, using simpler dot-product variants:
e_ij = sᵢᵀ hⱼ (dot) or e_ij = sᵢᵀ W hⱼ (general). No extra parameters for the dot variant, and the general variant uses a single learned matrix.

  • Placement: Bahdanau uses s_{i-1} (before decoding); Luong uses sᵢ (after)
  • Complexity: Bahdanau has a 2-layer score network; Luong is a single operation
  • Speed: Luong is faster at inference — dot products parallelize over the full sequence easily
  • Variants: Luong adds global vs. local attention (attending to a window instead of all positions)

In practice, performance is similar. Luong's simplicity made it the dominant form in later work, and both were superseded by the self-attention in Transformers.

03
How do attention weights relate to model interpretability? What are the limits of this view?
NLP Medium
›

The attention weight matrix [αᵢⱼ] shows which source positions the model attended to when generating each output token. For machine translation, a mostly-diagonal matrix suggests the model learned roughly monotonic alignment — a good sanity check. This made attention one of the first widely-used neural interpretability tools.

However, attention weights ≠ explanation. Several papers (notably Jain & Wallace 2019, Wiegreffe & Pinter 2019) showed:

  • Different attention patterns can produce identical outputs — attention is not unique
  • High attention on a token doesn't mean the model's prediction is "caused by" that token
  • Models can distribute attention uniformly and still make correct predictions
  • Adversarial attention distributions can be constructed that don't change the output

Better attribution methods (gradient-based: Integrated Gradients, SHAP) are generally more faithful. Attention is useful as a diagnostic and for visualization, but should not be treated as ground-truth feature importance.

Architecture
04
Why does the Transformer use positional encodings instead of relying on recurrence for sequence order?
Architecture Medium
›

Self-attention processes all tokens simultaneously — it has no inherent notion of order. Without positional information, "the cat sat on the mat" and any permutation of those words would produce identical representations.

The original Transformer (Vaswani et al., 2017) adds sinusoidal positional encodings to the token embeddings:
PE(pos, 2i) = sin(pos / 10000^{2i/d})
PE(pos, 2i+1) = cos(pos / 10000^{2i/d})

  • Fixed: no learned parameters; generalizes to sequence lengths not seen during training
  • Relative distances: the dot product between two sinusoidal encodings is a function of their distance, giving the model a signal about how far apart two positions are
  • Alternative — learned: BERT and GPT use learned position embeddings, which often outperform sinusoidal on fixed-length tasks but don't extrapolate
  • Modern — RoPE/ALiBi: rotary position embeddings and attention biases encode relative position directly in the attention score, extending context efficiently
05
What is multi-head attention and why use multiple heads instead of one large attention operation?
Architecture Hard
›

Multi-head attention runs h parallel attention functions on learned projections of Q, K, V, then concatenates and projects the results:
MultiHead(Q,K,V) = Concat(head₁, …, headₕ) Wᴼ
where headᵢ = Attention(Q Wᵢᴼ, K Wᵢᴷ, V Wᵢᵛ)

Why multiple heads?

  • Specialization: each head can attend to different relationship types — one might capture syntactic dependencies, another semantic similarity, another coreference
  • Avoiding averaging: a single attention head takes a weighted average of values; multiple heads in different subspaces can represent richer structure without collapsing to a single blur
  • Representational capacity: projecting to lower-dimensional subspaces (d/h per head) before attending is computationally cheaper than one full-dimension attention and often performs better

Empirically, analysis (Clark et al., 2019) shows different heads do specialize — some track subject-verb agreement, others attend to adjacent tokens or sentence-initial tokens. Pruning many heads has little effect, suggesting redundancy, but a diverse set matters.

06
When would you choose an encoder-only vs. decoder-only vs. encoder-decoder architecture?
Architecture Medium
›

Encoder-only (BERT-style): bidirectional — each token attends to all other tokens. Best for tasks where you need to understand the full input before producing an answer: classification, NER, QA (extractive), sentence embeddings. Doesn't generate text natively.

Decoder-only (GPT-style): autoregressive — each token only attends to past tokens (causal mask). Best for text generation, language modeling, few-shot prompting. The dominant architecture for large language models because scaling laws favor unified generation-centric training.

Encoder-decoder (T5/BART-style): encoder reads input bidirectionally; decoder generates output with cross-attention to encoder states. Best for conditional generation tasks where input and output differ structurally: machine translation, summarization, question generation.

  • NLU tasks (sentiment, classification) → encoder-only
  • Open-ended generation, chat, instruction-following → decoder-only
  • Translation, summarization, structured prediction → encoder-decoder

In practice, large decoder-only models (GPT-4, Claude) handle all three via prompting, making the distinction blurry at scale.

Training
07
Explain the vanishing gradient problem and how LSTMs address it.
Training Medium
›

In deep or long recurrent networks, gradients are backpropagated through time by repeated multiplication of Jacobian matrices. If those matrices have eigenvalues < 1, the product shrinks exponentially — gradients vanish before reaching early time steps, making it impossible to learn long-range dependencies.

The LSTM addresses this through its cell state c_t, which flows through time with additive updates rather than multiplicative ones. Three gates control information flow:

  • Forget gate f_t: what fraction of the previous cell state to keep: f_t = σ(W_f [h_{t-1}, x_t])
  • Input gate i_t: what new information to write to the cell
  • Output gate o_t: what to expose as the hidden state h_t

The key insight: when the forget gate is near 1, gradients can flow back through c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t almost unattenuated, because the gradient of c_t w.r.t. c_{t-1} approaches 1 (not the product of sigmoid derivatives). The cell state is a "gradient highway."

08
What is teacher forcing and what are its tradeoffs?
Training Medium
›

Teacher forcing feeds the ground truth token as input to the decoder at each step during training, rather than the model's own prediction from the previous step.

Pros:

  • Faster convergence — the decoder always sees correct inputs, so gradients are more stable
  • Avoids compounding errors during training (wrong predictions cascading into worse ones)

Cons — Exposure Bias: at inference time, the model must condition on its own (potentially wrong) outputs. It was never trained on this distribution, so small errors at early steps compound and the model can spiral into incoherent outputs.

Mitigations:

  • Scheduled sampling (Bengio et al., 2015): start with 100% teacher forcing; gradually decrease the probability during training so the model sees its own outputs increasingly often
  • Reinforcement learning fine-tuning: train with a sequence-level reward instead of token-level cross-entropy
  • Prof-forcing / contrastive objectives: match the joint distribution of teacher-forced and free-running outputs
09
Explain the difference between batch normalization and layer normalization. Why does the Transformer use layer norm?
Training Architecture Hard
›

Batch normalization normalizes each feature across the batch dimension: for feature j, compute mean and variance over all examples in the batch, then normalize. This stabilizes training by reducing internal covariate shift but creates a dependency on batch size and makes the stats unavailable for single-example inference.

Layer normalization normalizes across the feature dimension for each individual example: compute mean and variance over all features for a single token's representation, then normalize. No batch dependency.

Why LayerNorm for Transformers?

  • Sequence lengths vary — batches of sequences have different valid positions, making batch statistics unreliable for positions masked as padding
  • Autoregressive generation processes one token at a time (batch size = 1 at inference) — BatchNorm breaks in this regime
  • LayerNorm has no dependence on batch size and works identically at training and inference time

Modern variants: Pre-LN (normalize before the attention/FFN sublayer, not after) stabilizes training at large scale and is used in GPT-3 and most large LLMs.

Evaluation
10
What is BLEU score and what are its main limitations?
Evaluation Easy
›

BLEU (Bilingual Evaluation Understudy) measures n-gram precision between a hypothesis translation and one or more reference translations, with a brevity penalty to discourage excessively short outputs.
BLEU = BP · exp(Σ wₙ log pₙ)
where pₙ is clipped n-gram precision for n-grams of length 1–4.

Strengths: cheap, language-agnostic, reproducible, correlates with human judgment at the corpus level (not sentence level).

Limitations:

  • Measures surface overlap, not meaning — a synonym-based paraphrase can score 0 despite being correct
  • Poor correlation with human judgment at the sentence level
  • Sensitive to tokenization choices — scores are not comparable across toolkits without normalization
  • Reference-dependent — requires human-written references; one reference disadvantages divergent but correct translations
  • Doesn't penalize grammatical errors that happen to produce matching n-grams
11
How would you evaluate an MT model beyond BLEU?
Evaluation Medium
›

A multi-level evaluation suite for machine translation:

  • chrF: character n-gram F-score — more robust to morphological variation, less sensitive to tokenization
  • TER (Translation Edit Rate): minimum edit distance (insert/delete/substitute/shift) to transform hypothesis to reference, normalized by reference length — penalizes reordering
  • BERTScore: computes token-level cosine similarity between contextual embeddings of hypothesis and reference — captures semantic equivalence that n-gram metrics miss
  • COMET: a learned metric trained on human judgments (direct assessment scores); state-of-the-art correlation with human evaluation at segment level
  • Human evaluation: adequacy (does it convey the same meaning?) + fluency (is it natural-sounding?) — the gold standard but expensive

For attention-specific evaluation: alignment error rate (AER) compares model attention weights against gold-standard word alignments. A good attention model should produce low AER on language pairs with known alignment data.

12
What is perplexity and when is it a good (or bad) evaluation metric for language models?
Evaluation Medium
›

Perplexity is the exponentiated average negative log-likelihood of a test corpus under the model:
PPL = exp(−(1/N) Σ log P(xᵢ | x₁,…,xᵢ₋₁))
Intuitively, it measures how "surprised" the model is by the test data. Lower is better — a perplexity of 10 means the model is effectively choosing uniformly among 10 options at each step.

Good for:

  • Comparing models trained on the same data with the same tokenizer
  • Tracking training progress — perplexity reliably decreases as the model improves
  • Evaluating language model quality on in-distribution text

Bad for:

  • Cross-model comparison — perplexity depends on tokenization; a model with byte-pair encoding vs. character-level tokenization gives incomparable numbers
  • Downstream tasks — lower perplexity doesn't guarantee better task performance; a model can memorize surface statistics without understanding
  • Out-of-distribution evaluation — perplexity inflates dramatically on domains not seen during training, masking generalization quality
